RSA has a surprising property which is that you can find the corresponding message for a signature by using public information. This trick is sometimes called “backwards signing”.

In RSA encryption and decryption are inverses of each other and signing with RSA is “just” decryption (since decryption uses the private key exponent).

That is, to sign a hash, h: s = h^d mod n To verify the signature: h = s^e mod n

For backwards signing, you choose what you want the signature to be, and then you encrypt it to find the corresponding message.

This feature of RSA is generally considered useless since you can’t sign a message of your choosing but instead find the message for a corresponding signature of your choosing. This does mean though that a user can’t be trusted to be a key’s owner (know the private key) just because they can produce message, signature pairs.

Real RSA uses special padding and hashing for signatures to futher protect against this attack.

Challenge 10:

Many encryption schemes has a property knows an “malleability” where mathematical operations on encrypted messages have a predictable effect on the corresponding decrypted message.

For example, with XOR if you flip a bit in the ciphertext that corresponds to the same bit being flipped in the plaintext.

For RSA the malleable property is multiplication. That is, for two messages, m1 and m2:

enc(m1, e, n) * enc(m2, e, n) == enc(m1 * m2, e, n)

Stated more concretely, the encryption (or signature) of 15 is the same as the encryption (or sig) of 3 times the encryption (or sig) of 5.

In this challenge you are asked to generate a signature for a 256 bit hash. You are given the ability to sign smaller messages. Use this ability to construct a signature from the smaller factor’s signatures.

Challenge 11:

In this challenge a key has been specially generated to be weak to Pollard’s p - 1 factorization algorithm.

Pollard’s algorithm is efficient when one of its primes, p, is of a special form. That is, that p - 1 only has small factors, not big ones. One small example of this is the prime 17. Since 17 - 1 is 16 and 16 is a power of 2, 16 only has small factors.

Using Pollard’s p - 1 algorithm factor n to determine the private key and decrypt the provided encrypted message.

import math
import random

def gcd(a, b):
    while b:
        a, b = b, a % b
    return a

def pollards_p_1(n, B=10**6):
    a = random.randint(2, n-1)
    for j in range(2, B+1):
        a = pow(a, j, n)
        d = gcd(a - 1, n)
        if 1 < d < n:
            return d
    return None
